Gravity Does Arithmetic

Two circuits built from nothing but Newtonian point masses. First, one flyby used as a full half adder — gravity adds two bits. Then the milestone that makes ballistic logic a computer at all: two gates composed in series, the deflected output of one flyby flying on to become the input of the next.

01 · Half adder

1 + 1 = 10

The flyby is Fredkin–Toffoli's interaction gate. Both balls are input bits now. Deflected exits fire only when both flew — that's the carry (A AND B). Straight-through exits fire only when exactly one flew — that's the sum (A XOR B). Binary addition from one gravitational encounter.

02 · Two gates in series

A AND C = 1

Ball B always flies. Bit A gates the first flyby; bit C is a third ball aimed at B's post-gate-1 trajectory. B ends double-bent only if A bent it onto C's lane and C was there — a genuine AND, computed by composition. The aim can't come from billiard geometry: gravity has no shielding, so C's pull drags B off course during the whole approach, and the wire must be calibrated on the full three-body problem.

The chaos tax · measured

Nudge B's launch by 10−8: gate 1 grows the error ×6, gate 2 another ×17 — about 1 digit of launch precision spent per gate, with no restoring force to win it back. Float64 bankrolls ~12 gates at this scale. That decay curve is the research object.

No shielding, no insulation

Deflection falls off only as ~1/b (impact parameter), so idle trajectories still bend each other: the straight lane picks up a 7° crosstalk nudge from C's passing. Gates must be spaced so logic lanes diverge before wires cross — routing is part of the physics.

Why composition matters

One gate is a curiosity; gates that feed each other are a computer. The interaction gate plus routing is universal in the billiard-ball model, and gate 2 here consumes gate 1's output in flight — same trick, pure gravity. Numerics: DOP853, rtol 10−12, energy drift ≤ 7×10−13 across all runs.
